# 意思決定ログ

> 重要な判断を記録する。「なぜそうなったか」を残すことで、同じ議論を繰り返さない。
> 議事録から秘書が抽出して追記する。

## フォーマット

```
### [日付] 決定タイトル
- **決定**：何を決めたか
- **理由**：なぜそうしたか
- **却下した選択肢**：他に何を検討したか（あれば）
- **影響**：これにより何が変わるか
```

---

### 2026-05-09 秘書エージェントシステムの構築
- **決定**：Claude Code のサブエージェント機能（`.claude/agents/`）で秘書システムを構築する
- **理由**：追加ライブラリ不要・即使える・Markdown で管理できるシンプルな方法
- **却下した選択肢**：Claude Agent SDK / API でのフル実装（将来要件が増えたら再検討）
- **影響**：ユーザーは秘書とだけ会話し、秘書が調査・計画・実装担当に指揮する

### 2026-05-09 ナレッジベースの導入
- **決定**：`知識ベース/` ディレクトリで秘書の記憶を管理する
- **理由**：サブエージェントはセッションをまたいで記憶しないため、ファイルで代替する
- **却下した選択肢**：エージェント定義ファイルに直接書き込む（肥大化・管理困難になるため）
- **影響**：ユーザーが「覚えておいて」と言うたびに秘書がファイルを更新し、セッション開始時に読み込む

### 2026-05-09 議事録・意思決定ログの分離
- **決定**：議事録は `業務日誌/`（時系列）、意思決定ログは `知識ベース/`（テーマ別）に分ける
- **理由**：用途が異なる。議事録は「いつ何を話したか」、意思決定ログは「なぜそうなったか」
- **却下した選択肢**：同一ファイルにまとめる（検索性・可読性が下がるため）
- **影響**：秘書がセッション終了時に議事録を書き、重要決定を意思決定ログに抽出する
