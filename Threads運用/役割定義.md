# Threads運用チーム 役割定義

## アカウント基本設定

- **運用者**：kei（1人運用）
- **投稿頻度**：毎日（1日1投稿以上）
- **ターゲット**：27〜40歳の女性／会社員
- **テーマ**：NISA・保険見直し・節税・家計改善・住宅ローン・手当（社会保障/公的給付）
- **ボイス**：親しみやすく専門性のあるFP。断定や煽りは避け、選択肢を提示する語り口

## チーム構成

```
秘書
└─ Threads運用統括（ハブ）
    ├─ 企画担当（コンテンツ柱・カレンダー・お題出し）
    ├─ リサーチャー（FP特化／一次情報・統計・法令）
    ├─ ライター（Threads向け下書き）
    ├─ 校閲・コンプラ担当（金商法・景表法・誇大表現チェック）
    └─ アナリスト（投稿後の効果分析・改善提案・競合観察）
```

## 標準ワークフロー（1投稿あたり）

```
1. 企画担当
   → 月次の柱から今日のお題を1つ選定
   → 出力：投稿カレンダー/YYYY-MM.md にチェック

2. リサーチャー
   → お題に必要な数字・制度・出典を調達
   → 出力：ネタ帳/YYYY-MM-DD_お題.md

3. ライター
   → ネタ帳をもとにThreads投稿ドラフト作成
   → 出力：投稿ドラフト/YYYY-MM-DD.md

4. 校閲・コンプラ担当
   → 法令・誇大表現・出典抜けをチェック
   → 出力：投稿ドラフト/YYYY-MM-DD.md にOK or 修正指示を追記

5. kei が手動で投稿（Threadsアプリ）

6. アナリスト（週次／月次）
   → エンゲージメント分析・改善提案
   → 出力：分析レポート/YYYY-Www.md
```

## 各エージェントの守備範囲

| エージェント | やること | やらないこと |
|--|--|--|
| 企画担当 | コンテンツ柱設計、月次カレンダー、お題選定 | 本文ライティング、出典調達 |
| リサーチャー | 一次情報・統計・法令の出典付き調達 | 本文ライティング、お題決定 |
| ライター | Threads向け下書き作成 | 出典確認、コンプラ判断 |
| 校閲・コンプラ担当 | 法令・誇大表現・出典チェック | コンテンツ柱設計、新規ネタ調達 |
| アナリスト | 投稿後の数値分析、競合観察、改善提案 | 投稿生成、ネタ調達 |

## ハンドオフルール

- 各エージェントは「次の担当」が読みやすい形式で出力する
- 出典・数字・制度名は **必ず日付と出典URL付き**（古い情報の混入を防ぐ）
- 不明点は推測で埋めず「要確認」と明記する
